6 Idiosyncracies

annoying, but more or less hard to fix:

6.1 SVG HTML export size issue

If you export to html and embed an svg like this 

#+ATTR_ORG: :width 600
#+ATTR_HTML: :width 500px
file:printTabelle.svg

Or you could embed it specifically with explicit html, even an oversized version (but then you have to be smart about overflowing it correctly when screen width is small) 

<style>
.oversized {
margin-left: 50%;
transform: translateX(-50%);
}
</style>
<img class="oversized" src="printTabelle.svg" style="width: 150%"></img>

8 DONE Figure out how to not have a numbering for every latex environment at html export

Only those latex environments with a label tag, or equations should have a numbering.

Any starred environment won't have a label! If you want to typeset anything using latex, just put a starred dummy environment like so: 

#+LATEX_HEADER: \newenvironment*{orgfrag}{}{}

and then put this into the body of your org file: 

\begin{orgfrag*}
Hello
\end{orgfrag*}

If you want an svg instead of the usual png, use 

#+OPTIONS: tex:dvisvgm

at the top of your org file.
